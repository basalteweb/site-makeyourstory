FORMULAIRES
Les formulaires envoient les demandes via FormSubmit vers basalteweb@gmail.com.
Au premier test réel, FormSubmit demande généralement une validation sur l’adresse e-mail destinataire.
Avant de publier, teste une demande complète avec ton propre e-mail.

IMAGES
Lis images/IMAGES_A_AJOUTER.txt et dépose les fichiers WebP avec les noms exacts.
Les pages ne feront pas d’erreur si les images manquent, mais le rendu final dépendra directement de ces visuels.

ACCÈS PRIVÉ
access.html crée le portail d’entrée visuel demandé.
Le code DEMO-LOVE ouvre la démo.
Important : un site statique GitHub Pages + un code JavaScript ne constitue pas une protection réelle pour des photos privées. Pour de vrais espaces protégés, prévois plus tard une solution sécurisée avec authentification côté serveur ou un service spécialisé.

ANALYTICS / GTM
assets/analytics.js contient uniquement des identifiants de démonstration.
Ne les active pas avant d’avoir mis en place un mécanisme de consentement adapté. Les pages légales expliquent ce point.

VÉRIFICATIONS À FAIRE AVANT OUVERTURE DES VENTES
1. Vérifier dans mentions-legales.html : SIREN, RCS, adresse, régime de TVA et toute information juridique qui aurait changé.
2. Ajouter les coordonnées de ton médiateur de la consommation dans cgv.html avant de vendre à des particuliers.
3. Finaliser la durée d’hébergement / maintien du lien annoncée dans les offres.
4. Configurer la solution de paiement (par exemple Stripe) et adapter les CGV à la procédure réelle.
5. Remplacer les liens Instagram / TikTok fictifs du footer par tes liens.
6. Déposer les images WebP.
7. Configurer le domaine makeyourstory.fr sur GitHub Pages et vérifier HTTPS.

PAIEMENT STRIPE
MakeYourStory réutilise le même compte et la même solution Stripe que BASALTE-WEB. Les textes ont été ajoutés dans À propos, FAQ, mentions légales, confidentialité et CGV. Pour rendre les boutons de paiement actifs, il faudra ajouter les liens Stripe Checkout des produits Essentiel, Premium et Prestige.
